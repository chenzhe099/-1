方案建议采用“Spring Boot 业务后端 + Python AI 服务”，让业务逻辑与模型推理解耦。
选题三：云南特色农业智能诊断与生产管理平台
用户角色
农户：管理地块、作物和病虫害记录。
农技人员：审核识别结果和防治建议。
合作社管理人员：查看生产和市场风险。
管理员：管理模型、知识库和采集任务。
前端功能
农场、地块和种植档案管理。
病害图片上传及识别结果可视化。
农事日历和待办任务。
天气、病虫害和市场价格趋势图。
防治建议及规范原文对照。
模型性能、数据漂移和未知样本监控。
后端功能
图片异步上传、推理和结果通知。
农事任务生成、提醒和完成状态管理。
定时采集天气、市场价格和农业通报。
调用图像模型、RAG 和多 Agent 服务。
对未知病害进行拒识并进入人工审核队列。
维护模型、数据和知识库版本。
数据库设计
核心表包括：
users、farms、fields、crops、planting_cycles、observations、diagnosis_records、weather_records、market_prices、farming_tasks、knowledge_documents、model_versions、agent_runs。
图片存入 MinIO，关系数据库只保存文件地址、哈希值、识别结果和业务关联。
完整业务闭环
建立地块 → 登记作物 → 上传异常图片 → 模型识别 → RAG 检索农技规范 → Agent 综合天气与生育期 → 农技人员审核 → 生成农事任务 → 跟踪处置效果。
