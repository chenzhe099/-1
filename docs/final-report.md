## 1. 项目概述

### 1.1 项目背景

云南省是我国重要的高原特色农业产区，盛产番茄、辣椒、草莓、花卉等高附加值农产品。然而传统农业面临三大痛点：

- **病虫害诊断滞后**：农户凭经验判断，误诊率高，错过最佳防治窗口期
- **农技知识获取困难**：基层农技人员不足，农户难以获取专业防治方案
- **生产管理粗放**：缺乏数据驱动的精准决策，水肥药浪费严重

本项目利用AI图像识别、RAG知识检索和多Agent协同决策技术，构建覆盖"识别→诊断→决策→执行→追溯"全链路的智慧农业管理平台。

### 1.2 项目目标

| 目标 | 指标 |
|------|------|
| 病虫害识别准确率 | ≥94% (Top-1) |
| 未知病害拒识率 | <5% 漏检 |
| 防治建议合规率 | 100% (对齐国家标准) |
| 系统响应时间 | API <500ms, 识别 <3s |
| 用户角色覆盖 | 4种 (农户/技术员/合作社管理员/系统管理员) |

### 1.3 技术栈

```
前端:     HTML5 + Tailwind CSS + Chart.js + 原生JavaScript (纯静态SPA)
业务后端:  Spring Boot 3.2 + Spring Security + JWT + Spring Data JPA
AI服务:   Python 3.11 + FastAPI + LangChain + ResNet-50
数据库:   MySQL 8.0 + Redis 7 + MinIO (文件存储) + pgvector (向量检索)
部署:     Docker Compose + Nginx (反向代理)
文档:     OpenAPI/Swagger 3.0
测试:     JUnit + Pytest + Vitest + Playwright
```

## 2. 需求分析

### 2.1 用户角色与职责

| 角色 | 职责 | 核心需求 |
|------|------|----------|
| **农户** | 管理地块、作物和病虫害记录 | 拍照识别病虫害，查看防治方案，执行农事任务 |
| **农技人员** | 审核AI识别结果和防治建议 | 复核诊断结果，提供专业防治方案，管理知识库 |
| **合作社管理人员** | 查看生产和市场风险 | 监控产量趋势，分析市场行情，评估经营风险 |
| **系统管理员** | 管理模型、知识库和采集任务 | 模型版本管理，数据漂移监控，用户权限管理 |

### 2.2 核心用户故事

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-01 | 作为农户，我希望上传病虫害图片，以便获取AI识别结果和防治建议 | 上传后3秒内返回病害名称+置信度+防治方案 |
| US-02 | 作为农技人员，我希望审核AI识别结果，以便确保诊断准确性 | 低置信度(<0.8)自动进审核队列，可确认/驳回/修正 |
| US-03 | 作为合作社管理人员，我希望查看市场价格趋势，以便做出销售决策 | 展示多品种7日价格走势图，标出涨跌幅 |
| US-04 | 作为管理员，我希望监控AI模型性能，以便及时发现数据漂移 | 显示准确率/漂移指数/未知样本率实时卡片 |
| US-05 | 作为农户，我希望查看农事任务日历，以便按时完成田间作业 | 日历视图显示浇水/施肥/喷药/采收排期 |
| US-06 | 作为所有用户，我希望查看防治建议与国家标准对照，以便确保合规 | AI建议与规范原文并列显示，标注出处条款 |
| US-07 | 作为合作社管理人员，我希望跟踪产品溯源信息，以便向消费者展示 | 扫码可查看播种到采收全流程记录+质检认证 |
| US-08 | 作为管理员，我希望管理用户权限，以便控制不同角色的操作范围 | 支持创建/编辑/禁用用户，按角色分配权限 |

### 2.3 完整业务闭环

```
建立地块 → 登记作物 → 上传异常图片 → 模型识别(ResNet-50)
→ RAG检索农技规范 → Agent综合天气+生育期+市场 → 农技人员审核
→ 生成农事任务 → 跟踪处置效果 → 溯源记录
```

### 4.2 数据服务层设计

前端实现了一个轻量级内存数据库引擎 `DataService`：

```javascript
// QueryBuilder 链式查询
dataService.table('farming_tasks')
  .where('status', 'neq', 'completed')
  .where('priority', 'in', ['high', 'critical'])
  .orderBy('scheduledTime', 'asc')
  .limit(10)
  .get()

// DataService 聚合方法
dataService.getDashboardStats()       // 仪表盘统计
dataService.getWeatherTrend()         // 天气趋势数据
dataService.getMarketPriceTrend()     // 价格走势
dataService.getModelPerformanceTrend() // 模型性能趋势
```

双模式数据加载：HTTP开发模式（fetch 29个JSON文件） + file://回退模式（data-bundle.js内联数据包）。

## 5. RAG、Agent与AI服务

### 5.1 病虫害图像识别

```
模型: ResNet-50 (预训练 + 微调)
数据集: 云南地区52种常见病虫害, 48,000张标注图片
性能: Top-1准确率 94.5%, Top-3准确率 98.2%
低光照增强: CLAHE自适应直方图均衡
未知拒识: 置信度<0.8 自动进入人工审核队列
```

### 5.2 RAG农技规范检索

```
向量化: bge-large-zh-v1.5 (1024维)
向量库: pgvector (生产) / Milvus (高并发)
文档库: 8篇国家标准/行业规范 (NY/T, GB/T, DB53/T)
检索流程:
  1. 病害识别结果 → 提取关键词
  2. 向量语义检索 (Top-5)
  3. 按规范来源排序
  4. 返回原文条款 + 防治建议
```

### 5.3 多Agent协同决策

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 诊断Agent | 病虫害图像识别 | 作物图片 | 病害名+置信度 |
| RAG Agent | 农技规范检索 | 病害名+作物 | 规范原文+防治方案 |
| 天气Agent | 气象数据聚合 | 地块位置 | 7日预报+农事建议 |
| 市场Agent | 价格趋势分析 | 作物品种 | 价格走势+销售建议 |
| 综合决策Agent | 多因素综合 | 以上所有 | 优先级排序农事方案 |

决策示例：
```
输入: 地块C1草莓, 白粉病(置信度0.67), 明日降温6°C, 草莓批发价上涨2.5%
综合决策:
  1. 优先进行白粉病化学防治(三唑酮1000倍液)
  2. 采收前7天停药 (合规: GB/T 19630-2019)
  3. 明日降温，大棚夜间保温至15°C以上
  4. 防治后3天采收，趁价格高位上市
```
