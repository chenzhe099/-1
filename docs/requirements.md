## 1. 引言

### 1.1 项目背景

云南省是我国高原特色农业核心产区。传统农业生产中，农户凭经验判断病虫害，误诊率高达30%以上；农技知识获取依赖基层农技人员，覆盖率不足；生产管理以纸质记录为主，数据驱动决策能力薄弱。

本项目利用 Web 前端 + REST API + AI 图像识别 + RAG 知识检索技术，构建覆盖"地块管理 → 病虫害识别 → 农技规范对照 → 任务生成 → 执行跟踪"全链路的智慧农业管理平台。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| 统一农场管理 | 农场→地块→种植周期→田间观测 四级档案管理 |
| AI病虫害识别 | 图片上传、模拟识别、知识库匹配、防治建议 |
| 农事任务调度 | 自动生成/编辑/删除/排序/完成轮换 |
| 农技规范对照 | AI建议与国家标准原文并列对比 |
| 多角色权限 | 管理员/技术员/农户/合作社管理人员 四级权限 |
| 跨模块联动 | 病虫害→自动生成喷药任务，灌溉完成→土壤湿度联动更新 |
| 物联网监控 | 设备状态、远程控制、维护计划 |
| 数据可视化 | Chart.js 实时图表 + 数据驱动卡片 |
| 产品溯源 | 生产全过程记录 + 溯源码生成 |

### 1.3 适用范围

- 适用于云南高原特色农业产区（蔬菜、水果）
- 支持单个农场或多农场联合管理
- 适用作物：番茄、黄瓜、辣椒、茄子、草莓等

## 3. 用例描述

### 3.1 核心用例

```
┌─────────────┐     ┌─────────────────┐     ┌─────────────┐
│    农户      │────▶│  上传病虫害图片   │────▶│  AI识别引擎  │
└─────────────┘     └─────────────────┘     └─────────────┘
                           │                        │
                           ▼                        ▼
                    ┌─────────────┐         ┌─────────────┐
                    │ 识别结果展示  │         │ RAG规范检索  │
                    └─────────────┘         └─────────────┘
                           │                        │
                           └────────┬───────────────┘
                                    ▼
                           ┌───────────────┐
                           │ 规范原文对照    │
                           │ AI建议 vs 国标  │
                           └───────────────┘
                                    │
                                    ▼
                           ┌───────────────┐
                           │ 自动生成喷药任务 │
                           │ (syncModuleState)│
                           └───────────────┘
```

### 3.2 任务生命周期用例

```
新建任务 ──▶ [待开始] ──▶ [进行中] ──▶ [已完成] ──▶ 沉底折叠
                │                                    │
                └──────── [已取消] ◀──────────────────┘
                              │
                    ✅ 活跃任务数不变
                    (autoReplenishTask)
```

### 3.3 跨模块联动用例

```
病虫害记录(processing) ──syncModuleState()──▶ 地块状态→disease
                                                   │
                                   自动生成喷药任务(spraying/high)
                                                   │
任务标记完成(spraying/completed) ──▶ 地块soil_moisture +15%
                                                   │
                             所有病虫害已处理 ──▶ 地块状态→growing
```

## 4. 功能需求 (FR)

### FR-01: 登录与权限

| 字段 | 说明 |
|------|------|
| 优先级 | 高 |
| 描述 | 系统提供登录认证，4种角色各有不同的菜单可见性和操作权限 |

**功能点：**
- 演示账号：admin/123456（管理员），farmer/123456（农户）

- 角色-模块可见性矩阵：

| 角色 | 仪表盘 | 病虫害 | 农事 | 预测 | 管理 | 设备 | 溯源 | 权限 | 天气 | 市场 | 监控 |
|------|--------|--------|------|------|------|------|------|------|------|------|------|
| 管理员 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 技术员 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ | ✅ | ✅ | ✅ |

### FR-02: 仪表盘驾驶舱

| 字段 | 说明 |
|------|------|
| 优先级 | 高 |
| 描述 | 首页展示4个统计卡片（可点击跳转）+ 地块状态列表 + 农事任务 + 预警列表 |

**功能点：**
- 4个统计卡片：全部农事任务→农事决策 / 设备在线率→设备监控 / 预警信息→病虫害识别 / 预估产量→产量预测
- 地块状态列表：每个条目点击弹出综合详情（含任务、病虫害、观测、种植周期）
- 农事任务列表：活跃在前/已完成折叠，编辑/删除，排序规则（进行中+高优先>待开始>已完成沉底）
- 快捷操作按钮：点击跳转到对应模块

### FR-03: 病虫害识别

| 字段 | 说明 |
|------|------|
| 优先级 | 高 |
| 描述 | 图片上传→AI识别→知识库匹配→防治建议→规范对照 |

**功能点：**
- 拖拽上传 + 拍照识别（模拟模式）
- 识别历史：每条可点击查看详情（病害名/置信度/地块/时间/AI建议/症状/病因/防治方案）
- 知识库：搜索框即时过滤，卡片展示病害名+症状摘要+严重级别（中文：高/中/低）
- 规范原文对照面板：自动匹配最新识别记录对应的农技规范，展示AI建议vs国标原文
- 查看规范原文按钮：按病害名三级匹配（标题→关键词→模糊搜索）

### FR-04: 农事任务管理

| 字段 | 说明 |
|------|------|
| 优先级 | 高 |
| 描述 | 任务的全生命周期管理，支持增删改查和自动轮换 |

**功能点：**
- 任务创建：选择类型/地块/日期/时间/指派人/优先级
- 任务编辑：点击任务条目弹出编辑表单，可修改类型/状态/优先级/备注
- 任务删除：右侧×按钮，确认弹窗防误删
- 排序规则：进行中>待开始，同状态按优先级(高→中→低)，已完成/已取消沉底折叠
- 自动轮换：完成任务→自动从所有地块随机生成新任务补充
- 全部任务弹窗：detail()弹窗展示所有任务，支持编辑/删除/刷新
- 操作日志：每次CRUD自动记录到operation_logs

| 字段 | 说明 |
|------|------|
| 优先级 | 高 |
| 描述 | 模块间数据联动，操作一处自动更新关联模块 |

**联动规则：**
| 触发 | 效果 |
|------|------|
| 病虫害记录(processing) | → 地块状态=disease + 自动生成喷药任务(spraying/high) |
| 灌溉任务完成 | → 地块土壤湿度+15% |
| 病虫害全部已处理 | → 地块状态恢复growing |
| 有活跃任务 | → 休耕地自动变growing |
| 需要浇水的任务 | → 地块状态=watering |

## 5. 非功能需求 (NFR)
